# Interview Pocket

本地桌面面试复习工具 —— 从 Markdown 面试手册导入题库，围绕搜索、复习、错题、突击和掌握度统计形成闭环。

## 项目简介

Interview Pocket 是一个面向技术面试复习场景的本地桌面题库管理与复习工具。它支持从 Markdown 面试手册导入题库，并围绕搜索、复习、错题、突击和掌握度统计形成复习闭环。

- **本地优先**：所有数据保存在本地 SQLite 文件，不上传云端，不依赖后端服务。
- **面向个人**：为个人面试复习设计，不涉及多人协作或账号系统。
- **Markdown 导入**：将半结构化的 Markdown 面试手册解析为可管理的题库。
- **复习闭环**：从导入、检索、复习、统计到管理，覆盖面试复习全流程。

当前主要用于 AI 应用后端 / Java 后端 / 项目面试题复习资料管理。应用本身不接入 AI 模型。

## 为什么做这个项目

传统 Markdown / Word / 飞书文档复习存在几个问题：

- **题目多后难以检索**：155 道题以上时，Ctrl+F 搜索效率低，无法按相关性排序。
- **不知道哪些题复习过**：纯文档阅读没有状态跟踪，复习进度全凭记忆。
- **不知道哪些题掌握不好**：没有错题沉淀和掌握度统计，薄弱环节不清晰。
- **错题无法沉淀**：做错的题目散落在各处，无法集中复习。
- **面试前临时突击缺少独立模式**：面试前需要快速浏览，但普通复习流程太重。
- **多份题库来源难以管理**：同一主题多个版本的题库，无法按来源筛选和清理。
- **重复导入和旧版本题库难清理**：导入旧版本后需要手动去重。

Interview Pocket 用本地桌面工具把这些问题收敛为：**导入 → 检索 → 复习 → 统计 → 管理**。

## 功能特性

### 题库导入与管理

- Markdown 文件导入，自动识别题目结构
- 导入报告：解析数、新增数、重复跳过、答案统计
- 重复题目跳过（基于 title + source_key 去重）
- 来源筛选：使用路径哈希区分同名文件，界面仍显示原文件名
- 题库来源删除：按来源整份删除，同步清理复习数据
- 大题库支持：500 条本地加载上限，适配 155+ 道题规模

### 题库检索

- 关键词搜索，按相关性排序（标题 > 记忆锚点 > 答案 > 原始内容）
- 命中位置提示（标题 / 记忆锚点 / 标准答案 / 原始内容）
- 命中片段展示，快速定位匹配内容
- 来源筛选 + 复习状态筛选 + 搜索组合使用

### 复习模式

- 今日待复习：基于间隔复习规则自动计算
- 普通卡片复习：逐题查看答案，1-5 分自评打分
- 错题复习：集中复习错题次数 > 0 的题目
- 面试前突击模式：纯浏览模式，不写复习记录，支持不确定标记
- 复习完成后本轮统计：题数、平均分、最高分、最低分、低分题数、高分题数

### 复习统计

- 首页统计：总题数、今日待复习、今日已复习
- 已复习 / 未复习题数
- 错题数、低掌握题数（mastery_score < 60）
- 平均掌握度（仅已复习题目）
- 单题复习记录查看
- 题库列表掌握度进度条、复习次数、错题次数展示

## 技术栈

| 技术 | 用途 |
|------|------|
| Electron | 桌面应用框架，跨平台运行 |
| Vue 3 | 前端框架，Composition API |
| TypeScript | 类型安全 |
| electron-vite | 构建工具，主进程 / 预加载 / 渲染进程分离构建 |
| sql.js | 纯 JS SQLite 实现，无需 native 编译 |
| SQLite | 本地数据库，单文件持久化 |
| Vue Router | 前端路由 |

## 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                    Electron App                         │
├─────────────────────────────────────────────────────────┤
│  Renderer (Vue 3)                                       │
│  ├── 首页：统计概览、快速操作                               │
│  ├── 题库页：题目列表、搜索、来源筛选、复习状态筛选            │
│  ├── 导入页：文件选择、导入报告、题库来源管理                  │
│  ├── 复习页：卡片式复习、自评打分、复习记录                   │
│  ├── 错题复习：错题集中复习                                 │
│  └── 突击模式：纯浏览、不确定标记                            │
├─────────────────────────────────────────────────────────┤
│  Preload                                                │
│  └── contextBridge 安全暴露 API                          │
├─────────────────────────────────────────────────────────┤
│  IPC Layer                                              │
│  └── Renderer ↔ Main 双向通信                            │
├─────────────────────────────────────────────────────────┤
│  Main Process                                           │
│  ├── Markdown 文件读取和解析                              │
│  ├── SQLite 数据库操作（sql.js）                          │
│  └── IPC Handler：题库、搜索、复习、统计、来源管理           │
├─────────────────────────────────────────────────────────┤
│  Data Layer                                             │
│  └── %APPDATA%/interview-pocket/interview-pocket.db     │
└─────────────────────────────────────────────────────────┘
```

**数据流**：

```
Markdown 面试手册 → Electron dialog 选择文件 → Main 读取文件
→ Markdown parser 解析题目 → questions / review_progress 写入
→ 返回导入报告 → Renderer 展示报告
```

## 本地运行

### 环境要求

- Node.js 18+
- npm
- Windows / macOS / Linux 理论可运行，当前主要在 Windows 环境开发验证

### 命令

```bash
# 安装依赖
npm install

# 开发模式运行
npm run dev

# 构建生产版本
npm run build
```

### 数据位置

```
%APPDATA%/interview-pocket/interview-pocket.db
```

- 不上传云端：所有数据保存在本地 SQLite 文件。
- 不依赖 AI：纯本地运行，无需网络。
- 不依赖后端：无服务器，无账号系统。

## 使用流程

1. **准备 Markdown 面试手册**：按 `## Q1. xxx` / `## P0-1. xxx` 等格式组织题目。
2. **导入题库**：在导入页选择 Markdown 文件，查看导入报告。
3. **筛选题目**：在题库页通过来源、复习状态、关键词组合筛选。
4. **开始复习**：进入普通复习 / 错题复习 / 突击模式。
5. **查看统计**：首页查看复习进度、掌握度、错题数。

## Markdown 格式说明

**题目识别**：
- `## P0-1. 2 分钟介绍 xxx`
- `## Q1. 如果 xxx？`
- `## J1. 60 秒介绍 xxx`
- `## B1. 为什么不 xxx？`
- `## 1.1 请用 2 分钟介绍 xxx`

**答案识别**：
- `### 推荐回答` / `### 参考回答` → 标准答案
- `### 简短回答` → 简短答案
- `### 深入回答` / `### 详细回答` → 深入答案

**辅助信息**：
- `### 核心句` / `### 一句话锚点` → 记忆锚点
- `### 追问点` / `### 可能追问` → 追问列表
- `### 注意` / `### 禁忌说法` / `### 不要说` → 注意事项

## 项目边界

当前**不支持**：

- 云同步 / 登录注册 / 多端协作
- AI 自动生成题目 / AI 接入
- Markdown 富文本渲染（使用纯文本展示，降低 XSS 风险）
- 单题编辑 / 单题删除
- Windows 安装包发布 / 自动更新

这些是有意的架构选择，不是遗漏。
